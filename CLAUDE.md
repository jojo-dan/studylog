# studylog — 운영 룰 + AI 협업 가드레일

이 폴더에서 AI(Claude 등)와 함께 학습 진행할 때의 룰.
**학습 unit = 글감 unit.** done에 들어가는 게 곧 블로그 backlog.

## 구조

```
studylog/
├─ CLAUDE.md          ← 이 파일
├─ README.md
├─ roadmap.md         ← 학습 로드맵 + 글감 진행 순서 (SSOT)
├─ scratch.md         ← raw brain dump (날짜 헤더 누적)
├─ 00 inbox/          ← 모르는 거. 한 항목 = 한 .md
├─ 01 doing/          ← 지금 보고 있는 1~2개
└─ 02 done/           ← 정제 끝, 블로그 후보
```

## 운영 룰

- 한 unit = 한 글감 = 한 블로그 포스트
- 흐름: `00 inbox` → (호기심·필요 순) → `01 doing` → 풀고 정제 → `02 done`
- 폴더 이동만으로 상태 변화
- daily 의무 없음. 떠오르면 `scratch.md`에 날짜 헤더로 누적
- 5개 정도 `02 done` 쌓이면 발행처 결정

## 글감 작성 가이드

각 unit에 포함:

1. 질문 — 무엇을 모르는가
2. reference — 학습 자료 (출처 명시 = 글에서 인용 가능)
3. 내 프로젝트 적용 예시 — 본인 작업·프로젝트에 어떻게 쓰였는지
4. 정리 후 한 줄 요약 — done 이동 시 채움

## unit 크기 가이드

- 30분급: 정의·도구류
- 1~2시간급: 개념·구조
- 2~4시간급 이상: 시리즈로 분해 권장

## AI에게

### 학습 흐름 (한 unit 처리)

```
0. Owner: 00 inbox/X.md 선택 → 01 doing/으로 파일 이동
1. AI:    자료 출처 추천 (quick / 깊이 / 적용 예시 출처)
            └ 자료 본문 요약·풀이 금지. 출처 + 한 줄 설명만.
            └ 영어 자료 막히는 부분만 번역·해설은 OK
2. Owner: 자료 직접 읽기
3. Owner: 01 doing/X.md에 raw text 작성
            ├ 이해한 내용 (본인 언어)
            ├ 내 프로젝트 적용 예시
            └ 출처 (직접 인용 시 따옴표 + 출처)
4. AI:    사실 오류·누락 코멘트만 — 인라인 또는 "팩트 체크" 섹션
            └ 본문 직접 수정 금지 (덮어쓰기 금지)
5. Owner: 코멘트 보고 본인이 본문 수정
6. 4↔5 반복 — AI 코멘트 0건 또는 분해 신호 발생 시까지
7. Owner: 발행 톤 마무리 (제목 / 서론 / 결론)
8. AI:    마지막 팩트 체크 한 번 (사실 오류만)
9. Owner: 02 done/ 이동 결정
```

판정 권한: done 이동은 owner. AI는 코멘트·팩트체크만.

### 분해 신호 (단계 6에서 멈출 사유)

| 신호 | 처리 |
|------|------|
| 글감으로 너무 큼 | 1편 끝, 후속(2편/3편) 새 unit 분리 |
| 사전 지식 부족 | 트리거 A (아래) |
| 자료 부족 | 자료 찾기 새 task. 현 unit pause |
| 인지 피로 | 그날 stop, 다음 세션 |

추정 4시간 이상 unit은 **시작 전**에 시리즈로 분해.

### 로드맵 갱신 (AI 능동)

**트리거 A — 학습 중 사전 지식 부족 발견**

1. 사전 unit 식별
2. `00 inbox/<사전-unit>.md` 생성
3. `roadmap.md` 표 — 의존 unit 앞 행 삽입
4. 현 doing 파일 상단에 `## paused — 사전 unit #N 진행 중 (날짜)` 메모
5. 사전 unit을 `01 doing/`으로 이동

**트리거 B — scratch.md 새 메모에서 unit 후보 발견**

1. scratch 본문은 안 건드림
2. owner에게 제안 — "scratch의 X — unit 후보. roadmap 추가할까?"
3. owner confirm 후 → `00 inbox/`에 unit 생성 + `roadmap.md` 표 끝에 추가

### 기본 룰

- 새 unit 추가는 `00 inbox/`로
- doing → done 이동은 owner 결정 후. AI 자동 판정 금지
- `scratch.md`는 raw 영역. 자동 정리·요약·재구성 금지
- 글감 본문: 출처 + 적용 예시 둘 다 필수. 빠지면 done 미달
- **자료 풀이 금지** — AI는 자료 출처만. 풀이하면 owner 학습 가로챔
- **본문 직접 수정 금지** — AI 코멘트만. 본문은 owner가 직접 고침
